Description

        A chatbot is the short form of a chat robot, a computer program that simulates human conversation, or chat, through artificial intelligence. Typically, a chat bot will communicate with a real person. Chat bots are used in applications such as ecommerce customer service, call centers and Internet gaming. Chat bots used for these purposes are typically limited to conversations regarding a specialized purpose and not for the entire range of human communication.

        GoW will be the name of this chatbot and it is an acronym for Game of Words.

        This chat bot should be capable to maintain a conversation in English with a random person, answer his questions and it should be also capable to ask different questions based on their conversation.

        The project will be divided in 4 modules:

  • the implementation of the chatbot
  • building the knowledge base
  • text processing
  • testing and managing a web page with the project documentation

Modules

  1. Implementation Module

            This Module will describe a decision mechanism and altogether with the second and third modules will establish the communication interface and the resources format.

            In the implementation of GoW are used NodeJS for the interface and a python script for backend. This Python script will process the message sent by the client ( in this case, the client is the person which wants to talk with the chatbot) and will send back a response generated by his "brain".

  2. Knowledge Base Module

            This module will build the knowledge base, that will contain multiple domains, using an AIML database and an ontology. In the ontology will be memorized the general knowledge about the world like: the sky is blue, the water is wet.

            In AIML the pieces of data will be saved under the following format:

    <category> <pattern>What is your name?</pattern> <template>My name is Michael N.S Evanous.</template> </category> <category> <pattern>How old are you?<pattern> <template>![CDATA[I am <bot age=”age”/>.]]</template> </category>

            For each domain will exist an AIML file. The domains covered will be the ones highlited with green from the table from the following link http://www.alicebot.org/aiml/aaa/, but we will extend them in order to cover more pices of informations.

  3. Text Processing Module

            This module will process all texts like user input, external resources, output statements/restatement. The members of this module's team will correct the input errors, detecting the language, synonyms recognition, using synonyms, input pre-processing (name recognition, date recognition, phrase division into sentences).

  4. Testing and Documentation Module

            This module is the one which will manage a web page of the project which will contain a short description of the chatbot, all modules teams and all the tasks informations.
